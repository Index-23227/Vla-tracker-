# Data Format Guide

## Model YAML Schema

Each model is stored as a YAML file in `data/models/`.

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Model display name |
| `organization` | string | Research group / company |
| `date` | string | Publication date (YYYY-MM-DD) |
| `architecture` | object | Model architecture details |
| `benchmarks` | object | Benchmark results |

### Architecture Object

| Field | Type | Description |
|-------|------|-------------|
| `backbone` | string | Vision encoder |
| `llm` | string | Language model backbone |
| `action_head` | string | Action generation method |
| `action_space` | string | continuous / discrete |
| `parameters` | string | Model size |

### Benchmark Results

Each benchmark is a nested object:

```yaml
benchmarks:
  libero:
    libero_spatial: 84.2    # Success rate (%)
    libero_object: 88.4
    libero_goal: 72.0
    libero_long: 53.4
    source: "Table 1, original paper"
    date_reported: "2024-06-13"
```

- All scores are **percentages** (0-100)
- `source` is required — reference the exact table/figure
- `date_reported` tracks when this result was first published

## Benchmark YAML Schema

Benchmark definitions live in `data/benchmarks/`.

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Benchmark short name |
| `full_name` | string | Full paper title |
| `paper_url` | string | arxiv URL |
| `metric` | string | Primary metric name |
| `higher_is_better` | boolean | Sort direction |
| `categories` | object | Sub-benchmarks / task suites |
